## If you rebrand the plugin

When WordPress.org asks for a new distinctive name and slug, update these items together:

1. Plugin header `Plugin Name`
2. Plugin folder name
3. Main plugin file name if desired
4. `Text Domain`
5. `readme.txt` plugin title
6. Settings page labels and user-facing references
7. Any block names, text domains, and asset handles if needed

## Plugin Check reminder

WordPress Plugin Check expects the text domain to match the slug it sees for the submission. If the checker expects a different value than your code uses, the repository or submission slug may still be using the old plugin identity.
